# Доступные Агенты

Этот документ описывает доступные субагенты в системе, которые могут быть использованы для выполнения специализированных задач.

## Обзор

Система поддерживает два типа субагентов:

1. **general** - Общий агент для исследования сложных вопросов, поиска кода и выполнения многошаговых задач
2. **docs-writer** - Агент для создания и поддержки документации проекта

## Агенты

### general

**Описание:**  
Общий агент для выполнения разнообразных задач, включая исследование, поиск кода, веб-поиск и многошаговые операции.

**Возможности:**
- Исследование сложных вопросов
- Поиск и анализ кода в больших кодовых базах
- Выполнение многошаговых задач
- Работа с файлами и директориями
- Веб-поиск и анализ контента

**Когда использовать:**
- Для поиска конкретных функций или классов в коде
- При необходимости исследовать сложные технические вопросы
- Для выполнения комплексных задач, требующих нескольких шагов
- Когда требуется анализ больших объемов кода

**Пример использования:**
```
Используйте general агент для поиска всех вхождений функции `calculateTotal` в проекте.
```

### docs-writer

**Описание:**  
Специализированный агент для создания и поддержки технической документации проектов.

**Возможности:**
- Создание README файлов
- Написание API документации
- Составление руководств пользователя
- Форматирование и структурирование документации
- Поддержка различных форматов (Markdown, HTML)

**Когда использовать:**
- Для создания новой документации проекта
- При обновлении существующих документов
- Для стандартизации формата документации
- Когда требуется техническое письмо

**Пример использования:**
```
Используйте docs-writer для создания подробного README.md файла для нового проекта.
```

## Как использовать агентов

Агенты вызываются через специальный механизм системы. Для использования docs-writer агента требуется явный запрос от пользователя. General агент может быть использован автоматически для сложных задач.

## Ограничения

- Агенты работают в изолированной среде
- Каждый агент имеет ограниченное время выполнения
- Результаты агентов не сохраняются между сессиями
- docs-writer агент требует ручного вызова пользователем

## Лучшие практики

1. **Четкие инструкции:** Предоставляйте детальные описания задач для агентов
2. **Контекст:** Указывайте релевантный контекст проекта
3. **Ожидания:** Определяйте ожидаемый формат вывода
4. **Тестирование:** Проверяйте результаты работы агентов
